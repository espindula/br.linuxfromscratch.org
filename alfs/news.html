jhalfs Versão Estável 2.4 Lançada
Pierre Labastie - 2017/04/08

Depois de oito anos de lançamento contínuo, a equipe de desenvolvimento do jhalfs tem o prazer de anunciar o lançamento do jhalfs-2.4. Incontáveis mudanças foram feitas desde a última versão, para adaptação às novas versões dos livros LFS/CLFS/BLFS. Os principais novos recursos são a capacidade de usar um gerenciador de pacotes (desde que ele consiga usar instalação DESTDIR) e a capacidade de instalar automaticamente dependências de pacotes do BLFS.

Esse lançamento é compatível com os livros (B)LFS da série 7.x, bem como com as versões 8.0 recentes e o SVN atual, tanto Sys V quanto systemd. Também funciona com versões CLFS-3.x e desenvolvimento do CLFS.

Baixar tarball do jhalfs-2.4

Veja-se a lista dos livros suportados

Por favor, envie quaisquer comentários ou defeitos que encontrar para a lista de discussão alfs-discuss.

jhalfs 2.0 Lançado
Manuel Canales Esparcia - 2006/10/14

A equipe de desenvolvimento do jhalfs tem o prazer de anunciar o lançamento do jhalfs-2.0.

Novos recursos nessa versão:

  • Adicionado suporte para o livro de desenvolvimento do CLFS-2.0
  • Adicionado suporte para BLFS via estrutura blfs-tool
  • Adicionado suporte para instalar as dependências do blfs-tool enquanto se instalar sistemas xLFS
  • Um sistema de configuração completamente baseado em uma interface de menuconfig
  • Reescrita completa do Makefile para su para o(a) usuário(a) "lfs" e para entrar na jaula chroot somente uma vez.
  • Agora a construção do sistema precisa ser feita como um(a) usuário(a) sem privilégios, não mais como root. Isso significa que sudo agora é uma dependência exigida.
  • Verifique se o anfitrião tem as ferramentas adequadas instaladas e as versões capazes de executar jhalfs e construir o sistema.

O tarball do jhalfs-2.0 pode ser baixado a partir de http://www.linuxfromscratch.org/alfs/downloads/jhalfs/stable/jhalfs-2.0.tar.bz2

jhalfs 0.2 Lançado
Jeremy Huntwork - 2005/12/02

Existe uma nova implementação do ALFS disponível, chamada jhalfs. O principal propósito do jhalfs é o de fornecer uma pura construção de referência do LFS, extraindo os comandos para execução diretamente a partir do livro do LFS. Isso tem o benefício adicional de reduzir a manutenção, pois não existem perfis necessários.

Esse é o primeiro lançamento do jhalfs. A versão 0.1 foi uma prova do conceito e não um lançamento funcional. Você pode baixar a versão 0.2 aqui.

Quaisquer comentários ou perguntas concernentes ao jhalfs deveriam ser direcionadas para a lista de discussão alfs-discuss.

nALFS 1.2.5 Lançado
Thomas Pegg - 2005/10/10

A equipe do ALFS tem o prazer de anunciar o lançamento do nALFS versão 1.2.5. Essa é uma atualização de recursos/correção de defeitos para a versão 1.2.4, incorporando quatro mudanças:

1. Foi adicionado suporte para imprimir o nome de qualquer pacote que seja exigido, mas não presente, antes que o nALFS forneça a mensagem de construção abortada.

2. Uma nova opção de linha de comando foi adicionada para desabilitar a verificação de resumo (-d).

3. Uma nova opção de linha de comando foi adicionada para suportar baixar e verificar resumo de pacotes dentro dos elementos <unpack> e <download> sem executar as outras etiquetas em um perfil. (-D).

4. Corrigir adequadamente a detecção da biblioteca ncurses, se ncurses não for encontrado, o configure será resgatado. (Defeito 929)

Esse novo lançamento pode ser baixado a partir do local habitual.

Todos os comentários, perguntas e discussões desse lançamento deveriam ser direcionados para a lista de discussão alfs-discuss.
